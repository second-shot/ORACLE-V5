# Perplexity Operating Manual

## Rule
Use Perplexity as Oracle's research brain, not Oracle's memory.

## Structure
- one Space only
- one canon thread per pillar
- pinned strongest answers
- core files only
- no repeated restarts

## Mode discipline
- Best -> quick checks
- Pro Search -> serious architecture and comparison work
- Research -> milestone reports only
- Computer -> narrow execution tasks only

## Never waste spend on
- rediscovering the project from zero
- broad Computer tasks
- random brainstorming without canon
- mixing speculation and architecture in one thread
- uploading too much weak context

## Correct sequence
1. create Space
2. add custom instructions
3. upload canon files
4. run first planning prompt in non-Computer mode
5. pin strongest answer
6. break approved plan into small Computer tasks

## Space rule
The Space holds the canon.
Threads refine the canon.
Computer executes only narrow tasks.

## Constraint
Do not use Computer to think.
Use Computer to act.
