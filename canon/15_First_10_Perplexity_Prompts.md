# First 10 Perplexity Prompts

## Purpose
Use these inside the ORACLE V5 Space to shape the project without drift.

---

### 1. Irreducible identity
Using the files in this Space as canon, define the irreducible identity of ORACLE V5. Separate what is core, what is optional, and what should not be built.

### 2. Core architecture
Build the cleanest possible architecture for ORACLE V5 as a computer-first system. Include frontend shell, backend, object model, routing engine, archive layer, pricing engine, and agent orchestration. Show tradeoffs.

### 3. Universal object model
Create the universal object schema for ORACLE V5 so that every input can become a structured object. Include artists, works, drafts, projects, listings, memories, opportunities, proposals, agents, and archive items.

### 4. Routing engine
Define the routing engine for ORACLE V5. Show how objects move through capture, compression, classification, scoring, routing, generation, validation, storage, and resurfacing.

### 5. UI system
Define the full UI architecture for ORACLE V5: command surface, living home, decision board, archive atlas, artist pod, operator console, and proposal studio. Include states, modes, emotional tone, and interaction rules.

### 6. Artist modules
Create modular sub-paths for painter, singer, filmmaker, stylist, designer, and hybrid artist. Show what changes in each path and what stays universal in the core engine.

### 7. Pricing logic
Define the pricing and platform-routing engine for ORACLE V5 across eBay, Vinted, Depop, Facebook Marketplace, Etsy, Grailed, Vestiaire, Shopify, direct sale, consignment, and collector routes. Optimize for realistic outcome, not fantasy price.

### 8. Agent system
Define the bounded agent system for ORACLE V5. Include archive agent, pricing agent, listing agent, proposal agent, release agent, research agent, outreach agent, and operator agent. Specify limits, approvals, and logging rules.

### 9. Archive and memory
Design the archive and memory system for ORACLE V5. Separate live archive, cold archive, shadow archive, market archive, identity archive, and process archive. Show how resurfacing should work.

### 10. Roadmap
Create the phased build roadmap for ORACLE V5 from scratch. Phase 1 must prove the system is real. Show what to build first, what to postpone, and what would be fake depth.
