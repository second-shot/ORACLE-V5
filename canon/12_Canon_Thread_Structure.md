# Canon Thread Structure

## Purpose
Define the minimum thread structure inside the ORACLE V5 Space so the project does not drift.

## Rule
One thread per pillar.
Do not mix canon, speculation, and execution in the same place.

## Create these threads
- Oracle — Founding Identity
- Oracle — Core Architecture
- Oracle — Object Model
- Oracle — UI System and States
- Oracle — Pricing and Marketplace Routing
- Oracle — Agent Rules
- Oracle — Archive and Memory
- Oracle — Roadmap / First Build Slice

## How to use them
- keep one pillar per thread
- refine inside the same thread instead of restarting
- pin the strongest answer in each thread
- treat pinned answers as working canon until replaced

## Constraint
Do not spawn random new threads for the same question.
Continuity saves credits and preserves structure.
