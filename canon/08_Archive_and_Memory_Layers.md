# Archive and Memory Layers

## Purpose
Define how ORACLE V5 stores, recalls, and resurfaces value.

## Rule
The archive is not a graveyard.
Memory is not a transcript dump.

## Core layers
- Live Archive
- Cold Archive
- Shadow Archive
- Market Archive
- Identity Archive
- Process Archive

## Live Archive
Active material that still has current relevance.

## Cold Archive
Older material kept for record, context, and future reference.

## Shadow Archive
Forgotten, unfinished, or undervalued material with possible future value.

## Market Archive
Works, objects, and projects that are sale-ready or commercially usable.

## Identity Archive
Key works and materials that define the artist or operator.

## Process Archive
Sketches, drafts, references, experiments, and work-in-progress context.

## Memory functions
- store structured objects
- link related works and ideas
- resurface similar material
- detect hidden value
- prevent repeated thinking
- preserve evolution over time

## Example queries
- show similar works
- show dormant ideas linked to this
- show unfinished work that fits this project
- show pieces with market potential
- show references matching this tone

## Constraint
Archive must support retrieval, routing, and future action — not just storage.
