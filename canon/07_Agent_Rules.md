# Agent Rules

## Purpose
Define the bounded specialist agents inside ORACLE V5.

## Rule
Agents are tools, not gods.
They assist, propose, rank, draft, and route.
They do not act with hidden autonomy.

## Core agents
- Archive Agent
- Identity Agent
- Pricing Agent
- Listing Agent
- Project Agent
- Proposal Agent
- Release Agent
- Research Agent
- Outreach Agent
- Operator Agent

## What agents can do
- organise
- classify
- compare
- draft
- propose next actions
- ask for missing information
- rank options
- generate structured outputs

## What agents cannot do
- perform hidden high-risk actions
- overwrite canon without approval
- silently change commercial terms
- execute irreversible actions without review
- invent false certainty

## Required rules
- every agent has one clear role
- every agent has limits
- every agent leaves logs
- every important output is inspectable
- high-value or irreversible actions require approval

## Output style
Agents should return:
- recommendation
- reasoning
- missing info
- next step

## Constraint
The agent system must increase clarity and execution, not create chaos or fake intelligence theatre.
