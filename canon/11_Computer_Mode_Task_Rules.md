# Computer Mode Task Rules

## Purpose
Define how ORACLE V5 should use Perplexity Computer mode without wasting credits or losing structure.

## Rule
Computer mode is for execution, not discovery.
Do planning first in Ask / Pro Search / Research.
Use Computer only when the plan already exists.

## Use Computer mode for
- turning an approved roadmap into a task board
- creating structured implementation plans
- comparing 2–3 locked options into one decision memo
- generating clean spec documents from approved canon
- building spreadsheets or structured tables from existing decisions
- completing one narrow action with one clear output

## Do not use Computer mode for
- broad brainstorming
- rediscovering the product vision
- vague architecture exploration
- repeating context already stored in the Space
- open-ended ideation with no success condition

## Prompt rule
Every Computer prompt must contain:
- one task
- one output
- one success condition
- one scope boundary

## Good Computer prompt shape
- Take the approved Oracle canon in this Space
- Do one specific thing
- Output it in one specific format
- Stay inside one specific scope

## Example tasks
- turn the roadmap into an implementation plan
- compare Electron vs Tauri for Oracle and end with one recommendation
- convert pricing logic into a decision table
- build a module list for the first real version
- create a build spec for one surface of the UI

## Constraint
If the task can be answered with normal search and synthesis, do not spend Computer mode on it.

## Core principle
Use cheap modes to think.
Use Computer mode to execute.
